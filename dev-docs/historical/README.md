Historical Design Documents

This directory contains archived design explorations, early proposals, and closed V1 process records for Astichi.

Early / superseded design docs:

- `AstichiApiDesignProposal.md` — original API design proposal.
- `AstichiApiDesignComposed.md` — early composed-form design exploration.
- `AstichiV1Milestones.md` — the original milestone breakdown for V1 implementation. Superseded by the addendum design docs and by `../V2Plan.md` / `../V2ProgressRegister.md`. Preserved for original rationale.

Closed V1 process docs (frozen at V1 exit):

- `V1StartHere.md` — V1 process entry point.
- `V1Plan.md` — V1 execution plan.
- `V1ProgressRegister.md` — final V1 progress state.
- `V1DeferredFeatures.md` — tracker of features deferred from V1. Two entries (bind-external and loop unroll) were later reinstated as V2 scope; the remaining entries stay deferred.
- `v1_issues/` — V1 issue log (open + resolved).

These documents are preserved for historical context and to track design rationale, but they do not reflect the current implementation constraints or active API.

Active specifications and process:

- For the normative design, see `../AstichiApiDesignV1.md` and its addendums (`-InsertExpression`, `-BindExternal`, `-UnrollRevision`).
- For the active process, see `../V2StartHere.md`, `../V2Plan.md`, `../V2ProgressRegister.md`.
